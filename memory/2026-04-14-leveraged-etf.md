assistant: [[reply_to_current]] 一般来说，**大多数 Java PDF 识别/解析库本身不会调用外部接口**，通常是：

- **本地解析 PDF 结构**
- **本地提取文本、图片、坐标、字体**
- 如果带 OCR，也是**本地算法 / 本地模型 / 本地依赖**

也就是说，像这些常见库通常都是**纯本地处理**：
- **Apache PDFBox**
- **iText**（偏生成/解析）
- **OpenPDF**
- **Tabula-java**（表格抽取）
- **Tika**（封装提取）
- **Tess4J / Tesseract 接 OCR**（本地 OCR）

## 但要注意 3 个例外

### 1. 你自己接了云 OCR / AI 服务
比如代码里调用：
- 阿里云 OCR
- 腾讯云 OCR
- Google Vision
- OpenAI / Claude / Azure Document Intelligence

那当然就不是纯本地。

### 2. 某些商业 SDK 可能带联网授权/遥测
不是帮你识别内容上传云端，
但可能会有：
- license 校验
- 遥测
- 更新检查

这个要看厂商实现。

### 3. “识别” 和 “解析” 不是一回事
- **解析型库**：直接读 PDF 里的文本层，不需要联网
- **OCR 型库**：如果 PDF 是扫描件，就要靠 OCR
  - 本地 Tesseract 这类，还是不联网
  - 云 OCR 才会联网

## 简单结论
**如果你用的是常见 Java 开源 PDF 库，默认理解就是本地解析，不会偷偷调用外部接口。**
