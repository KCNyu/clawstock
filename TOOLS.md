# Rick's Stock Analysis Tools

## Portfolio Analyzer

我为 kcn 创建了一个专属的股票分析系统。

### 文件说明
- `portfolio.json` - kcn 的持仓数据
- `stock_analyzer.py` - 主分析脚本
- `.api_keys` - API 密钥（不要提交到 git）
- `check_portfolio.sh` - 快速查看脚本

### 使用方法

```bash
python3 stock_analyzer.py
```

## 数据源清单（按优先级）

> 服务器在**境外**，CN 系 API（新浪/腾讯）会 403。每次更新数据时按下表顺序自动尝试。

### 港股行情

| 优先级 | 来源 | 方式 | 状态 | 备注 |
|--------|------|------|------|------|
| 1 | **东方财富** | `push2.eastmoney.com` 批量 API | ✅ 稳定 | 无需 key，支持批量，境外可访问 |
| 2 | **Yahoo Finance** (`yfinance`) | `fast_info` 批量 | ⚠️ 限速 | 频繁调用会被 429，冷却后恢复 |
| 3 | **腾讯财经** | `qt.gtimg.cn/q=r_hkXXXXXX` | ❌ 境外 403 | 仅国内 IP |
| 4 | **新浪财经** | `hq.sinajs.cn/list=r_hkXXXXX` | ❌ 境外 403 | 仅国内 IP |

**港股代码格式：**
- 东方财富：`116.02208`（6位，补零）
- yfinance：`2208.HK`（不补零 + `.HK`）
- 腾讯：`r_hk002208`（6位补零）
- 新浪：`r_hk02208`（5位补零）

## 踩坑记录

- **2026-03-11**：发现港股数据一直是 3.10 的旧数据，原因是 Finnhub/Sina/Tencent 对境外 IP 全部 403。改用**东方财富批量 API** 解决。

- 新浪/腾讯财经 API 对境外 IP 完全封锁，服务器不在国内就别用。
